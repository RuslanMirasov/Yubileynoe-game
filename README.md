# Yubileynoe Game

## О проекте

`Yubileynoe Game` — браузерная мини-игра, в которой игрок управляет корзиной, ловит печеньки и избегает врагов.

Основной игровой цикл:

- раунд длится `60` секунд;
- на поле равномерно появляются печеньки и враги;
- печеньки нужно ловить;
- врагов лучше не ловить;
- в конце показывается popup с результатом;
- если собраны все печеньки, показывается победная всплывашка;
- если собраны не все, показывается popup проигрыша.

Проект построен без фреймворков. Вся логика находится в `assets/js`, стили — в `assets/scss`, страницы — в корне проекта.

## Инициализация приложения

Точка входа — [assets/js/scripts.js](./assets/js/scripts.js).

Он запускает:

- `initPreloader(PRELOADER_DELAY)`
- `initPopup()`
- `initGame(INIT_GAME_DELAY)`

Логика по шагам:

1. Прелоадер ждёт `window.load` и через задержку добавляет `body.loaded`.
2. Popup-модуль поднимает `window.popup`.
3. Игра проверяет, что на странице действительно есть игровая разметка.
4. После задержки стартует новый раунд.

## JS-навигация

### `preloader`

Файл: [assets/js/preloader.js](./assets/js/preloader.js)

Публичный метод:

- `initPreloader(delay = 0)`

Что делает:

- ждёт полной загрузки страницы;
- через заданную задержку добавляет класс `loaded` на `body`.

Зачем это нужно:

- запуск анимаций страницы;
- скрытие прелоадера;
- синхронизация визуального старта игры.

### `popup`

Файл: [assets/js/popup.js](./assets/js/popup.js)

Публичный API:

- `window.popup.open(id)`
- `window.popup.close()`

Как работает:

- `open(id)` закрывает все `.popup`, затем открывает popup с нужным `id`;
- `close()` снимает класс `active` со всех popup.

Ожидаемая разметка:

- popup должен иметь класс `.popup`;
- popup должен иметь уникальный `id`.

Примеры:

```js
window.popup.open('success');
window.popup.open('try-again');
window.popup.close();
```

## Игра: ключевые фичи

### 1. Константы

Файл: [assets/js/game/constants.js](./assets/js/game/constants.js)

Это главный файл настройки раунда.

Что здесь лежит:

- скорость движения корзины;
- базовая скорость падения;
- боковые отступы генерации;
- множитель ускорения;
- длительность раунда;
- количество печенек;
- количество врагов;
- список падающих объектов и их HTML-разметка.

Если нужно изменить баланс, обычно начинать надо именно с этого файла.

### 2. State

Файл: [assets/js/game/state.js](./assets/js/game/state.js)

`createGameState()` создаёт единый объект состояния раунда.

Что хранится в `state`:

- координаты корзины;
- флаги управления;
- `requestAnimationFrame` id для всех циклов;
- время старта игры;
- очередь спавна;
- индекс следующего предмета;
- список активных предметов;
- счёт игрока;
- остаток печенек;
- флаг завершения игры.

Важно:

- `state.myPoints` — текущее количество пойманных печенек;
- `state.targetPoints` — сколько печенек ещё осталось собрать;
- эти значения являются источником правды для UI.

### 3. Управление корзиной

Файл: [assets/js/game/basket/move.js](./assets/js/game/basket/move.js)

`initBasketMove(...)` отвечает за:

- перемещение корзины мышью/тачем;
- управление стрелками клавиатуры;
- ограничение движения по ширине поля;
- синхронный сдвиг передней и задней части корзины.

Возвращает контроллер с методами:

- `destroy()`
- `updateBounds()`
- `renderBasket()`
- `setBasketByClientX()`

`destroy()` обязателен при рестарте, чтобы не наслаивались обработчики событий и `raf`.

### 4. Сканирование зоны ловли

Файл: [assets/js/game/basket/scan.js](./assets/js/game/basket/scan.js)

`initBasketScan(...)` отвечает за горизонтальную проверку попадания предмета в зону корзины.

Как это работает:

- берётся прямоугольник `data-target-scanner`;
- для каждого активного предмета считается горизонтальное пересечение;
- если пересечение больше `50%`, предмет получает внутренний флаг `isInBasketZone = true`.

Важно:

- сканер нужен как геометрическая зона ловли, а не как декоративный элемент.

Возвращает:

- `destroy()`
- `updateAllItemsZoneState()`
- `updateItemZoneState()`

### 5. Генерация предметов

Файл: [assets/js/game/items/spawn.js](./assets/js/game/items/spawn.js)

`initItemsSpawn(...)` отвечает за:

- подготовку полного списка предметов на раунд;
- расчёт времени появления;
- расчёт позиции по `x`;
- рендер HTML каждого предмета;
- запуск появления предметов по расписанию.

Особенности текущей логики:

- поток предметов распределяется на весь `GAME_TIME`;
- последним предметом всегда будет печенька;
- ускорение применяется ступенчато, каждые `5` предметов;
- предметы не генерируются ближе `CANVAS_SIDE_PADDING` к краям;
- спавн равномерный, а не пачками после падения предыдущих объектов.

Возвращает:

- `destroy()`

### 6. Падение и поимка предметов

Файл: [assets/js/game/items/fall.js](./assets/js/game/items/fall.js)

`initItemsFall(...)` отвечает за:

- хранение активных предметов;
- движение предметов вниз;
- удаление предметов после выхода за пределы поля;
- проверку момента поимки;
- привязку пойманного предмета к корзине по оси `x`.

Логика поимки:

- у предмета есть флаг `isInBasketZone`, который обновляет сканер;
- когда нижняя граница предмета пересекает линию ловли, принимается решение;
- если в этот момент предмет в зоне корзины, он считается пойманным;
- после этого предмет получает класс `is-captured`;
- по `y` он продолжает движение вниз;
- по `x` он уже движется вместе с корзиной.

Это решение защищает от бокового подхвата предмета после прохождения линии ловли.

Возвращает:

- `destroy()`
- `registerItem()`

### 7. Типы предметов

Файл: [assets/js/game/items/types.js](./assets/js/game/items/types.js)

Содержит семантические типы:

- `plus`
- `minus`

Используются в очках и звуках.

### 8. Очки

Файл: [assets/js/game/score/points.js](./assets/js/game/score/points.js)

`initPoints(...)` отвечает за бизнес-логику очков.

Что делает:

- инициализирует начальные значения в `state`;
- связывает UI очков с `state`;
- реагирует на поимку `plus` и `minus`;
- запускает `pointToast(...)`;
- вызывает звуки поимки.

Логика:

- при поимке печеньки `state.targetPoints` уменьшается, `state.myPoints` увеличивается;
- при поимке врага, если у игрока уже есть очки, `state.myPoints` уменьшается, а `state.targetPoints` увеличивается.

Важно:

- `state` обновляется сразу;
- задержки применяются только к DOM-отрисовке через `pointToast`.

### 9. Анимация очков

Файл: [assets/js/game/utils/pointToast.js](./assets/js/game/utils/pointToast.js)

Отвечает за анимацию перелёта очков между банками и за задержанную отрисовку чисел.

Публичный API:

- `initPointToast(...)`
- `pointToast(type)`
- `window.pointToast(type)`

Поддерживаемые типы:

- `plus`
- `minus`

Как работает:

- числа в банках берутся из `state`;
- все элементы `[data-my-points]` обновляются синхронно;
- все элементы `[data-target-points]` обновляются синхронно;
- для `plus` анимация летит из `data-target-bank` в `data-my-bank`;
- для `minus` — наоборот.

Если на странице несколько элементов `[data-my-points]` или `[data-target-points]`, они все получают актуальные значения.

### 10. Звуки

Файл: [assets/js/game/utils/sound.js](./assets/js/game/utils/sound.js)

Звуки построены на `AudioContext`.

Что делает модуль:

- один раз предзагружает и декодирует аудиофайлы в память;
- хранит их в кэше `AudioBuffer`;
- на каждый play создаёт новый `AudioBufferSourceNode`;
- поэтому один и тот же звук может играть несколько раз подряд без ожидания окончания предыдущего.

Поддерживаемые звуки:

- `fall.mp3`
- `catch.mp3`
- `success.mp3`
- `game-over.mp3`

Публичные методы:

- `initGameAudio()`
- `playCatchSound(scoreType)`
- `playSuccessSound()`
- `playGameOverSound()`

Логика:

- `fall + catch` воспроизводятся при поимке печеньки;
- `fall` воспроизводится при поимке любого предмета;
- `success` играет при открытии победной всплывашки;
- `game-over` играет при открытии проигрышной всплывашки.

### 11. Утилиты

Файлы:

- [assets/js/game/utils/clamp.js](./assets/js/game/utils/clamp.js)
- [assets/js/game/utils/rect.js](./assets/js/game/utils/rect.js)

Назначение:

- `clamp(value, min, max)` — ограничение значения диапазоном;
- `getClientRect(element)` — обёртка над `getBoundingClientRect()`;
- `getHorizontalIntersectionRatio(itemRect, zoneRect)` — расчёт горизонтального пересечения.

## Финал раунда

Финал контролируется в [assets/js/game/index.js](./assets/js/game/index.js).

Раунд считается завершённым, когда одновременно выполнены три условия:

1. прошло `GAME_TIME`;
2. все предметы уже были заспавнены;
3. активных предметов на поле больше нет.

После этого:

- если `state.targetPoints === 0`, открывается popup `success`;
- иначе открывается popup `try-again`.

Также воспроизводится соответствующий финальный звук.

## Перезапуск игры

Перезапуск доступен двумя способами:

- `window.game.restart()`
- кнопка с атрибутом `data-restart-game`

Что делает рестарт:

- закрывает popup;
- останавливает старые `raf`-циклы;
- снимает старые обработчики;
- удаляет активные предметы;
- создаёт новый `state`;
- запускает новый раунд.

## Что менять в типичных задачах

Редактировать:

- [assets/js/game/constants.js](./assets/js/game/constants.js)

Чаще всего меняются:

- `GAME_TIME`
- `COOKIES_SUM`
- `ENEMYS_SUM`
- `FALL_SPEED`
- `END_SPEED_MULTIPLIER`

### Изменить правило поимки

Редактировать:

- [assets/js/game/items/fall.js](./assets/js/game/items/fall.js)

Именно там находится момент принятия решения:

- пойман предмет или нет;
- когда выставляется `isOut`;
- когда предмет становится `is-captured`.
